# Ethereum Devnet

This repository provisions a minimal Ethereum devnet that runs a local Geth execution client and a Teku consensus client against a deterministic genesis generated by [ethpandaops/ethereum-genesis-generator](https://github.com/ethpandaops/ethereum-genesis-generator).

## What you get

- **Execution + consensus** – Geth exposes JSON-RPC on `8545`, WS on `8546`, Engine API on `8551`, and discovery/peer traffic on `30303`.
  Teku exposes the Beacon REST API on `4000` and validator/libp2p ports on `9000`.
- **Deterministic genesis** – Genesis artifacts are produced from `values.env`, which pins the chain ID, fork versions, validator count (64), and timing values.
  - `just clean-genesis` will set `GENESIS_TIMESTAMP` to now, so the chain starts immediately.
- **Validator material** – The `keygen` container generates a 64-validator keystore from the mnemonic embedded in `values.env`.
  Teku mounts those keys from `eth-devnet-genesis/keystore/teku-keys` and starts proposing/attesting automatically.
- **Prefunded execution accounts** – The chain ships with prefunded accounts defined upstream in [`genesis_constants.star`](https://github.com/ethpandaops/ethereum-package/blob/main/src/prelaunch_data_generator/genesis_constants/genesis_constants.star).
  Any account listed there can be used immediately for testing without needing faucet funds.

## Repository layout

- `values.env` – Source of truth for every knob that feeds the genesis generator (preset, fork epochs, validator balances, premine list, etc.).
  Update this before rerunning `just genesis` if you need different parameters.
- `docker-compose.yml` – Orchestrates four services: `keygen`, `genesis`, `geth-init`/`geth`, and `teku`.
  Genesis data and keys are generated once and reused on subsequent starts.
- `eth-devnet-genesis/` – Output from the genesis tooling (metadata, configs, beacon state, keystores, `jwt/jwtsecret`).
  Persist it if you want deterministic restarts; delete it to regenerate from scratch.
- `geth-data/`, `teku-data/` – Execution and consensus data directories.
  Removing them wipes chain history/state.

## Prerequisites

- [Docker](https://docs.docker.com/engine/install/) (version 20.10.0 or above to include `compose`)
- [just](https://just.systems/)
- ~5 GB free disk space for client data and logs

## Usage

Run `just` to see a list of commands and descriptions, or read the [justfile](./justfile).

1. Adjust anything you need inside `values.env` (fork versions, validator counts, premine map, etc.).
2. Start the network:
   ```bash
   just up reset
   ```
   The first start may take a minute while keys and genesis artifacts are created.
3. Interact with the network using your preferred tools:
   - Execution RPC: `http://localhost:8545`
   - Execution WS: `ws://localhost:8546`
   - Engine API: `http://localhost:8551` (JWT secret at `eth-devnet-genesis/jwt/jwtsecret`)
   - Beacon REST API: `http://localhost:4000`
   - Libp2p (Teku/Geth): `9000/30303`
4. Stop the network:
   ```bash
   just down
   ```
5. Restart the network, using existing devnet setup and chain data:
   ```bash
   just up
   ```

### Resetting the devnet

Stop everything and delete generated state:

```bash
just up reset
```

If you want to delete only specific volumes:

```bash
# Docker volumes:
# ethereum-docker_eth-devnet-genesis
# ethereum-docker_geth-data
# ethereum-docker_teku-data
# Example:
docker volume rm ethereum-docker_eth-devnet-genesis
```

### Tips

- To add more prefunded accounts, edit `EL_PREMINE_ADDRS` inside `values.env` (JSON map) or update the upstream constants file before generating genesis.
- Teku's fee recipient defaults to `0x8943545177806ED17B9F23F0a21ee5948eCaa776` (`values.env::WITHDRAWAL_ADDRESS`).
  Change it if you need different payout behavior.
- You can run only a subset of services (e.g., `docker-compose up geth teku`) once the genesis artifacts exist.
